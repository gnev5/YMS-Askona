# YMS-14: Дополнительные столбцы в отчёте сверки ТЛ

## Статус
Review

## Источник
Запрос из Telegram-группы YMS-Askona-dev от 2026-07-20.

## Требование
После выполнения/открытия сверки ТЛ пользователь должен иметь возможность при просмотре отчёта добавить в таблицу дополнительные столбцы с данными по строкам/записям, а не видеть только статус, ТЛ, строку файла и ID бронирования.

## Принятое поведение
- В карточке результата сверки появляется блок «Дополнительные столбцы».
- Доступные столбцы строятся из данных строк, которые уже возвращает API:
  - `file_data` — сохранённые колонки Excel по строке файла;
  - `yms_data` — данные найденной записи/записей YMS.
- Пользователь может включать/выключать столбцы чекбоксами без перезапуска сверки.
- Основные колонки отчёта остаются: статус, номер ТЛ, строка файла, ID бронирования.

## План проверки
- Frontend source-regression: страница результата содержит управление дополнительными столбцами и использует `file_data`/`yms_data`.
- Frontend build/typecheck при доступном Node.js.
