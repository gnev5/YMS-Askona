# YMS-14: Дополнительные столбцы в отчёте сверки ТЛ

## Статус
Review

## Источник
Запрос из Telegram-группы YMS-Askona-dev от 2026-07-20.

## Требование
После выполнения/открытия сверки ТЛ пользователь должен иметь возможность при просмотре отчёта добавить в таблицу дополнительные столбцы с данными по строкам/записям, а не видеть только статус, ТЛ, строку файла и ID бронирования.

## Принятое поведение
- В карточке результата сверки появляется выпадающий список «Выбрать столбцы» с чекбоксами.
- Доступные столбцы ограничены полями, которые используются в таблице «Мои бронирования»: дата, время, док, объект, тип ТС, поставщик, зона, тип перевозки, кубы, статус YMS, пользователь.
- Данные берутся из `yms_data` найденной записи/записей YMS; произвольные поля Excel и служебные поля API в список не выводятся.
- Пользователь может включать/выключать столбцы без перезапуска сверки.
- Основные колонки отчёта остаются: статус сверки, номер ТЛ, строка файла, ID бронирования.

## План проверки
- Frontend source-regression: страница результата содержит выпадающий список дополнительных столбцов и ограничивает список полями из «Мои бронирования».
- Frontend build/typecheck при доступном Node.js.
