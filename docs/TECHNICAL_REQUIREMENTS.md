# Техническое задание на разработку системы YMS (Yard Management System)

## 1. Общие сведения

### 1.1. Наименование системы
Полное наименование: Система управления складским двором (Yard Management System)
Сокращенное наименование: YMS

### 1.2. Назначение и цели создания системы
Система предназначена для автоматизации процессов управления погрузочно-разгрузочными работами (ПРР) на складских территориях компании Askona. Система позволяет планировать, контролировать и оптимизировать движение транспортных средств на территории складского комплекса.

**Основные цели создания системы:**
- Оптимизация использования погрузочно-разгрузочных доков
- Сокращение времени простоя транспортных средств
- Повышение эффективности работы складского персонала
- Обеспечение прозрачности процессов ПРР для всех участников
- Снижение количества ошибок при планировании и выполнении ПРР

## 2. Требования к системе

### 2.1. Функциональные требования

#### 2.1.1. Управление пользователями
- Регистрация и аутентификация пользователей
- Разграничение прав доступа (роли: администратор, перевозчик)
- Управление профилями пользователей

#### 2.1.2. Управление доками
- Создание, редактирование и удаление доков
- Настройка параметров доков (название, статус, размеры, грузоподъемность)
- Определение типа дока (универсальный, вход, выход)
- Привязка доков к зонам

#### 2.1.3. Управление типами транспортных средств
- Создание, редактирование и удаление типов ТС
- Настройка длительности обслуживания для каждого типа ТС

#### 2.1.4. Управление рабочими расписаниями
- Создание расписаний работы доков по дням недели
- Настройка рабочего времени и перерывов
- Установка пропускной способности доков

#### 2.1.5. Управление временными слотами
- Автоматическая генерация временных слотов на основе расписаний
- Просмотр доступных временных слотов в календарном представлении
- Фильтрация слотов по датам

#### 2.1.6. Управление бронированиями
- Создание бронирований на ПРР
- Указание информации о транспортном средстве и водителе
- Выбор поставщика, зоны и типа транспорта
- Отмена и редактирование бронирований
- Просмотр истории бронирований

#### 2.1.7. Управление справочниками
- Управление зонами
- Управление поставщиками
- Управление типами транспорта

### 2.2. Нефункциональные требования

#### 2.2.1. Требования к производительности
- Время отклика системы при стандартных операциях не должно превышать 2 секунды
- Система должна поддерживать одновременную работу не менее 100 пользователей
- Время генерации временных слотов не должно превышать 30 секунд для периода в 1 месяц

#### 2.2.2. Требования к безопасности
- Аутентификация пользователей с использованием JWT-токенов
- Шифрование паролей пользователей
- Защита от основных типов атак (XSS, CSRF, SQL-инъекции)
- Логирование действий пользователей

#### 2.2.3. Требования к интерфейсу
- Интуитивно понятный пользовательский интерфейс
- Адаптивный дизайн для работы на различных устройствах
- Календарное представление для визуализации временных слотов
- Модальные окна для создания и редактирования объектов

#### 2.2.4. Требования к интеграции
- REST API для взаимодействия с внешними системами
- Документация API с использованием Swagger

## 3. Архитектура системы

### 3.1. Общая архитектура
Система построена на основе микросервисной архитектуры с использованием контейнеризации (Docker). Основные компоненты:
- Фронтенд: React + TypeScript + Vite
- Бэкенд: FastAPI (Python)
- База данных: PostgreSQL

### 3.2. Структура базы данных
Основные сущности:
- Пользователи (users)
- Доки (docks)
- Типы транспортных средств (vehicle_types)
- Рабочие расписания (work_schedules)
- Временные слоты (time_slots)
- Бронирования (bookings)
- Зоны (zones)
- Поставщики (suppliers)
- Типы транспорта (transport_types)

## 6. Требования к инфраструктуре

### 6.1. Требования к серверному оборудованию
- Процессор: не менее 4 ядер
- Оперативная память: не менее 8 ГБ
- Дисковое пространство: не менее 100 ГБ

### 6.2. Требования к программному обеспечению
- Docker и Docker Compose
- PostgreSQL 16 или выше
- Nginx или аналогичный веб-сервер для проксирования запросов
