# План работ по добавлению управления бэкапами

## 1. Backend

- [ ] Создать новый роутер `backups.py` в `backend/app/routers/`.
- [ ] В `backups.py` реализовать следующие эндпоинты:
    - [ ] `GET /api/backups/`: Получение списка всех файлов из папки `backups/`.
    - [ ] `POST /api/backups/create`: Запуск скрипта `backup_database.py` для создания нового бэкапа. Команда должна выполняться асинхронно.
    - [ ] `DELETE /api/backups/{filename}`: Удаление указанного файла бэкапа из папки `backups/`.
    - [ ] `POST /api/backups/restore/{filename}`: Запуск скрипта `restore_database.py` для восстановления из указанного бэкапа.
    - [ ] `POST /api/backups/upload`: Загрузка файла бэкапа в папку `backups/`.
    - [ ] `GET /api/backups/download/{filename}`: Скачивание указанного файла бэкапа.
- [ ] Зарегистрировать новый роутер в `backend/app/main.py`.

## 2. Frontend

- [ ] Найти компонент, отвечающий за главное меню (предположительно в `frontend/src/App.tsx` или похожем файле).
- [ ] Создать новый раскрывающийся список "Администрирование".
- [ ] Переместить существующий пункт "Пользователи" в новое меню.
- [ ] Добавить в "Администрирование" новый пункт "Бэкапы", который будет вести на `/admin/backups`.
- [ ] Создать новую страницу `frontend/src/pages/AdminBackups.tsx`.
- [ ] На странице `AdminBackups.tsx` реализовать:
    - [ ] Запрос к `GET /api/backups/` для получения и отображения списка бэкапов.
    - [ ] Для каждого файла в списке отобразить его имя и кнопки:
        - [ ] "Скачать" (вызывает `GET /api/backups/download/{filename}`).
        - [ ] "Удалить" (вызывает `DELETE /api/backups/{filename}`).
        - [ ] "Восстановить" (вызывает `POST /api/backups/restore/{filename}`).
    - [ ] Добавить кнопку "Создать новый бэкап" (вызывает `POST /api/backups/create`).
    - [ ] Добавить форму с полем `input type="file"` и кнопкой "Загрузить" для загрузки бэкапа (вызывает `POST /api/backups/upload`).
- [ ] Добавить новый маршрут в систему роутинга приложения для страницы `/admin/backups`.

## 3. Проверка

- [ ] Проверить, что в главном меню появился новый пункт "Администрирование".
- [ ] Проверить, что "Пользователи" и "Бэкапы" находятся внутри "Администрирование".
- [ ] Проверить, что страница "Бэкапы" открывается и корректно отображает список файлов.
- [ ] Проверить работу всех кнопок: создание, загрузка, скачивание, удаление, восстановление.
- [ ] Убедиться, что после создания, загрузки или удаления бэкапа список на странице обновляется.
